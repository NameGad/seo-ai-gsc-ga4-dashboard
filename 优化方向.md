我觉得这个项目未来可以发展成一个 **本地化 SEO Intelligence 工作台**，核心不是“多做几个图表”，而是沉淀你自己的 SEO 数据资产，然后让 AI 基于历史数据做分析、诊断、预测和行动建议。

我建议方向是：

```text
GSC 数据
+ GA4 数据
+ 本地历史数据库
+ AI 分析层
= SEO 决策系统
```

**未来功能方向**
1. **GSC 深度分析**
   - 页面衰退监控 ✅ 已完成
   - 关键词排名波动 ✅ 已完成
   - 低 CTR 机会 ✅ 已完成
   - 页面 cannibalization 关键词内耗 ✅ 已完成
   - Query intent 自动分类 ✅ 已完成规则版，后续可升级 AI 分类
   - 按国家、设备、搜索类型拆分 ✅ 已完成抓取与本地存储，重新 Load Data 后生效
   - 新增 / 流失关键词检测 ✅ 已完成
   - 按页面类型拆分 Collection / Product / Blog / Other ✅ 已完成

2. **GA4 接入**
   - Landing page sessions
   - Engagement rate
   - Conversions / key events
   - Source / medium
   - Revenue 如果有电商数据
   - 页面点击后质量分析

3. **GSC + GA 联动**
   - GSC 点击上涨，GA engagement 有没有上涨
   - GSC 排名提升，但转化是否变差
   - 高点击低转化页面
   - 低点击高转化页面
   - SEO 流量价值，而不只是点击量

4. **本地数据资产**
   - 自动定时同步
   - 每天保存快照
   - SQLite 长期结构化存储
   - 后面加 DuckDB 做大规模分析
   - 可导出 CSV / Excel / Markdown 报告

5. **AI 分析**
   - 自动总结本周 SEO 变化
   - 解释异常波动
   - 生成优化任务清单
   - 给页面标题 / Meta 建议
   - 关键词分组和意图聚类
   - 生成老板/客户可读的 SEO 周报

**我建议的优化清单**
**阶段 1：数据底座稳定化**
1. 给 SQLite 增加版本化 migrations。✅ 已完成
2. 把当前 `node:sqlite` 换成更稳定的 SQLite 依赖，避免 experimental warning。
3. 增加数据备份机制，例如 `data/backups/seo-data-日期.sqlite`。✅ 已完成
4. 增加 URL 规范化表，统一处理：
   ```text
   trailing slash
   http / https
   query params
   canonical URL
   language / country path
   ```
5. 增加站点维度表：
   ```text
   sites
   properties
   data_sources
   sync_jobs
   ```
   ✅ 已完成 `sites`、`data_sources`、`sync_jobs` 基础表，`properties` 可在 GA4 接入时扩展。

**阶段 2：GSC 抓取能力增强**
1. 支持批量同步所有 GSC properties。
2. 支持按维度抓取：
   ```text
   date
   page
   query
   page + query
   country
   device
   searchAppearance
   ```
3. 支持 search type：
   ```text
   web
   image
   video
   news
   discover
   googleNews
   ```
4. 增加增量同步：只抓最近 N 天。
5. 增加 row pagination / startRow，尽可能拿更多数据。
6. 增加数据质量提示，因为 GSC API 本身不保证返回全部行，更多是 top rows。

**阶段 3：GA4 接入**
1. OAuth scope 加：
   ```text
   https://www.googleapis.com/auth/analytics.readonly
   ```
2. 新增 GA4 property 列表。
3. 新增 GA4 数据表：
   ```text
   ga_daily_metrics
   ga_landing_pages
   ga_channels
   ga_events
   ga_conversions
   ```
4. 新增 GA 页面：
   - traffic overview
   - landing pages
   - channels
   - events
   - conversions

**阶段 4：GSC + GA 联动分析**
1. 建立 `landing_page` 统一维度。
2. 把 GSC page 和 GA landingPage 做 join。
3. 做核心分析模型：
   ```text
   clicks -> sessions
   impressions -> clicks -> engaged sessions
   query -> landing page -> conversion
   ```
4. 做页面机会分层：
   - 高曝光低 CTR
   - 高点击低互动
   - 高排名低转化
   - 低点击高转化
   - 排名下降但转化价值高

**阶段 5：AI 分析层**
1. 先做规则型 AI 报告，不急着上向量库。
2. 增加 AI 分析 API：
   ```text
   /api/ai/weekly-report
   /api/ai/page-diagnosis
   /api/ai/query-clustering
   /api/ai/opportunities
   ```
3. 给 AI 准备结构化上下文：
   ```text
   site
   date range
   top pages
   rising queries
   declining queries
   low CTR pages
   GA engagement
   conversion impact
   ```
4. 后面再加 embeddings：
   - query 聚类
   - 页面语义相似
   - 历史报告检索
   - AI 问答：问“最近 rapoo 哪些页面 SEO 下滑？”

**阶段 6：产品体验**
1. 增加全局站点切换。
2. 增加保存视图和筛选条件。
3. 增加任务系统：
   ```text
   title 优化
   content update
   internal link
   technical issue
   conversion review
   ```
4. 增加报告导出：
   ```text
   Markdown
   PDF
   CSV
   Excel
   ```
5. 增加 dashboard 首页，显示所有站点的健康度。

**我建议下一步先做**
优先做这 3 个：

1. **SQLite 数据层稳定化 + migrations**
2. **GSC 历史趋势页面** ✅ 已完成，已按 property 分组并增加重复快照去重
3. **GA4 OAuth scope 和 property list 接入**

原因是：这三步做完以后，你的数据基础、历史查看、GA 接入入口就都稳了，后面 AI 分析不会悬空。

参考依据：GSC Search Analytics API 支持按维度查询，但有内部限制，不保证返回全部数据行；GA4 Data API 的 `runReport` 是按 dimensions/metrics 返回表格化报告；OpenAI embeddings 适合搜索、聚类、异常检测等语义分析场景。
